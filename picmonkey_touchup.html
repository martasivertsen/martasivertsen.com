Touch Up Mobile

Lead UX Design & Research . Project Management . 2017

One of PicMonkey's most popular photo editing tools is Touch Up. This tool on picmonkey.com makes super complicated photoshop tricks, a breeze and ease to pick up. It was a no brainer to add this feature for the PicMonkey mobile app. I was the design lead for this project working closely with product and development to launch an even easier Touch Up tool for mobile.

The challenge

After the launch of the PicMonkey Mobile app we quickly discovered the biggest missing in the app for our current users was the Touch Up feature for mobile.

The goal was to fast follow the launch of the app with the most loved Touch Up features on .com. In addition the business hoped to use this opportunity monetize the app by making Touch Up a pay to play feature. Lastly, we saw this as an opportunity to improve the desktop Touch Up experience for mobile and apply advanced technologies like face recognition.

In tandem with dev, I explored out how multiple colors and gradients could work with face recognition.
With the input of dev I explored gestures and controls needed for each Touch Up effect.
How would Touch Up work compared to Adjustments or Effects? One of the biggest design challenges was fitting this new technology into the current app's tool language. It was important to reuse controls found in other editing tools, so this new feature felt familiar and easy to dive into.

Defining the feature set

After diving into the possibilities of the tech with dev and understanding how Touch Up would fit into our current tool language, I finalized the feature set and must haves to take this to market. Touch Up on mobile would have the most popular tools found on picmonkey.com, utilize face recognition, and would allow users to apply and adjust effects in a single session.

Details about kicking off the project by diving deeper into the mobile touch up market. Describe the recommendations I gave to the team of must have feature lists in order to be in parity with competitors, but where we could set the product apart.

Touch Up on PicMonkey mobile would include the ability to adjust the intensity of all effects applied via face recognition. Competitors that did not offer this left the user with little options to adjust the effect based upon the user.
Touch Up on PicMonkey mobile would include a way to auto apply the effect via facial recognition, but it would also allow users to manually apply the effect for full customization.
Touch Up on PicMonkey mobile would also include a way to preview where the effect was applied after face recognition. The effects can be subtle and a bolder way to preview where it's being applied is important.
Lastly, we would use indicators that flash after tap or drag in order to help educate how to use the various tools that require varying gestures.

Visualizing those features

As development was fleshing out the technology based on early brainstorm sessions, I began to finalize the UI and edge cases that were emerging.

Gathering user feedback

We did one round of adhock usability tests before finalizing the product. Our target users were current PicMonkey users that occasionally or regularly use touch up their photos on a mobile device. I wanted to get the entire team involved in user testing, so had a dev pair with me during each study to take notes and observe. This was super valuable in getting the entire team on the same page and aware of the usability concerns.

It’s really subtle and I’m not sure that something happened.

For the test when users tapped on the face recognition tool to auto apply the effect there was no obvious feedback that something changed on the photo. We added an animation that played after each tap of the auto application, to help indicate that yes something did change. In addition the icon shifted color with each tap to indicate the intensity increase.

I’m not sure about auto applying the effect, I want to pick and choose where to apply it.

Half the users we tested with liked that we auto applied the effect when tapping on a tool, but the other half wanted both, an auto application and a way to just manually apply the effect. We decided to separate the auto application into a tool called Presto, that would turn on the auto application, but tapping into a tool only would be for manually brushing on the effect only.

I want it to move more with my finger. I can barely see what’s being done.

Nip Tuck, specifically "Melt", was a tool that perplexed all of the users we tested with. It wasn't clear what was happening and how to use it. After the study, I paired with dev to adjust the technology to be more sensitive and cause more dramatic effects. We also added a touch indicator to help establish "Reduc" and "Fill" effects of Nip Tuck as tap only effects, where "Melt" was a paint only tool. Lastly we changed the name of the tool from "Melt" to "Nudge" to help indicate the outcome of the effect.

Key takeaways

In the end this feature was really fun and creative to use, that was just the right amount of complexity balanced with ease of use. As our first monetized component in the app, we learned that we have higher conversions if we put a hard paywall in front of a paid feature vs. offering an option to demo and then purchase. Fortunately, we discovered this quickly after launch and fast followed with an updated release.

Image.net Redesign

User Research . UX Design Lead . UI Design . 2014

Image.net is a site owned by Getty Images that provides free publicity photos and videos to media clients. The site distributes content provided by industry clients, via a complex internal facing portal. I was tasked with leading the UX effort to improve the overall user experience and address specific client concerns. In addition, I completed a visual overhaul providing suggestions for user interface improvements.

The challenge

The current site had some challenging navigational hierarchies and required a significant amount of time to onramp new users. The main user goals that we wanted to target and improve upon were being able to move images between folders faster, view folder permissions more easily, and scroll through images quickly to then approve or deny media access.

Understanding the user

After interviewing our primary users to dive into how they were currently using the site and it's pain-points, we established two user types and their workflows.

A challenging part of this redesign was any changes made would need to benefit both workflows. One workflow involved approving a batch of assets upfront and then releasing to the media, while the other workflow required releasing individual assets to then approve on a case by case basis.

Gathering user feedback

For this project, we knew it would be important to get early feedback from users, so we followed an iterative design model, with each design followed by a set of user interviews.

Study goals

  1. Continue to gather data around how our users work.
  2. Which design concept simplifies the user workflow.

Study questions

  1. Can you walk us through how you use image.net? Do you search, do you spend more time on the asset detail or folder level, what are some key actions you take regularly on assets?
  2. What are your general impressions with each design?
  3. You’ve just created a new project and uploaded assets, now how would you set permissions, view asset details, or publish content with each design?
  4. Anything else that can make these designs work better for you?

Design round one

For the first round of interviews we presented three design solutions to a small set of users. All users tested were site admins that used the site on a daily basis. The designs tested focused on improving folder access so assets could be moved from folder to folder more easily, providing a way to scroll through images for quicker approval or denial, and quicker access to folder permissions.

I like being able to see the folder permissions easily, but figuring out how to get to them might be difficult.

We confirmed that providing folder permissions was highly desirable, but it needed to be more clear how they could be accessed. A simple expand or close control was not enough.

It’s really nice to be able to see the image thumbnails with the large image preview and folder structure at the same time.

We confirmed that viewing image details was just as important as viewing folder structure for quick drag and drop. The challenge was to figure out how to equally represent both without overwhelming the user.

I think this design would be too cumbersome and difficult to click through these images.

Though this design is attempting to provide a way to scroll through images, it does not fulfill the need to be able to move through them quickly and approve or deny access to each image.

Design round two

For the second round of interviews we expanded the number of study participants, narrowed the design concepts down to two, and further refined the designs to help solve for the primary concerns.

There’s a lot going on with this design and so many different levels, it’s starting to become difficult to find my place.

The benefit and downfall of this design was that we were attempting to display the entire folder structure, from the top product categories to the final assets. This was useful, but also provided a lot of information for the user to parse. We confirmed that assets and folders were high priority than product navigation.

It’s super useful to have the image preview on the right, though I still want to be able to see the image at a much larger size.

Users were leaning more towards having easy access to image details versus folder structure and their permissions. They were getting a larger image preview than what we had previously, but it needed to be larger in order to properly view the image.

Applying the visual layer

As we moved from wireframes to visual design, I started to define a visual language based on the current structure, but with a modern, fresh take. The main goal was to update the visuals through flat ui elements and larger more touchable links to improve navigation.

The applied color palette with updated greys.
Set of icons that where more tapable to improve the mobile web experience.

Finalizing the design

The final design included access to the folder structure with a toggle to switch between contents and folder permissions. Instead of an inline image preview a full screen image preview was included in order to zoom and analyze the image. This new view also allowed the user to scroll through a series of photos and approve or deny along the way. In addition, we collapsed the previous left sided product list into a top level menu.
